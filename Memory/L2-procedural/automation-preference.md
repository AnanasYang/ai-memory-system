# 自动化偏好模式

在多个项目和日常工作中表现出强烈的自动化倾向，看到重复工作时第一反应是设计脚本或工具来解决。

---

## 模式定义

### 核心特征
- **重复工作识别**：快速识别可自动化的重复任务
- **工具化思维**：偏好用脚本/代码/配置替代人工操作
- **流程固化**：将临时解决方案固化为可复用的工作流
- **监控意识**：自动化任务配套日志、告警、状态检查

### 触发场景
- 定期重复的数据处理（日报、周报、数据同步）
- 多步骤的手动操作流程（部署、发布、检查）
- 跨系统数据搬运（Jira→飞书、GitHub→本地）
- 定时触发的工作（cron、heartbeat、定时任务）

### 典型表现
1. 看到手动填表 → 想写脚本自动填充
2. 看到定期同步 → 想配置定时任务
3. 看到多步骤部署 → 想做成一键脚本
4. 看到状态检查 → 想做自动化监控

## 边界与例外

### 适用条件
- 重复频率 > 每周 1 次
- 步骤明确、可编码
- 有明确的触发条件（时间/事件）

### 不适用场景
- 一次性、探索性工作
- 需要人工判断的创意性任务
- 步骤模糊、经常变化的流程
