# 记忆系统初始化事件

## 背景

Bruce 提出希望建立一套可迭代的 AI 记忆系统，用于提炼和记录：
- 价值观、性格特征、心智模式
- 思维模式、工作习惯、行为偏好
- 决策原则、关注方向

最终目标：让 AI 更好地与 Bruce 合作，甚至模拟其决策。

## 讨论内容

### 1. 记忆系统架构设计
- 确认使用 **5 层记忆架构**（L0-L4）
- 划分 **Intent（主动）** 和 **Memory（被动）** 双轨道
- 建立 **Meta 元数据层** 记录系统迭代

### 2. 各层定义与沉淀规则
- **L0 状态层**：当前会话上下文，10轮自动摘要
- **L1 情境层**：近期对话摘要，每周 review，30天归档
- **L2 行为层**：重复行为模式，每月 review
- **L3 认知层**：思维模式与决策框架，每季度 review
- **L4 核心层**：价值观与身份，**仅人工修改**

### 3. 触发机制设计
- **定时触发**：Cron 每周/月/季度提醒
- **条件触发**：Heartbeat 检测模式满足时
- **主动触发**：用户指令触发

### 4. 文件格式与存储
- 使用 **Markdown + YAML Frontmatter**
- 独立目录：`ai-memory-system/`
- 与日常工作目录分离，保持纯净

## 产出成果

1. ✅ 完整目录结构（Memory/Intent/Meta/archive）
2. ✅ 7 个模板文件（含 L4 identity.md）
3. ✅ 4 个 Cron 定时任务
4. ✅ AGENTS.md 更新（记忆系统专门章节）
5. ✅ HEARTBEAT.md 配置（7项检查任务）
6. ✅ QMD 索引建立（ai-memory collection）

## 关键决策

| 决策 | 内容 |
|------|------|
| 文件格式 | Markdown + YAML Frontmatter |
| 存储位置 | `ai-memory-system/` 独立目录 |
| L4 修改权限 | **仅 Bruce 人工修改，AI 不得自动更新** |
| Review 频率 | L1每周、L2每月、L3每季度、L4每年 |
| 归档策略 | L1 30天后归档保留，其他层长期保留 |

## 观察到的行为模式（L2 候选）

1. **系统化思维** — 要求清晰的层级结构和规则
2. **边界意识强** — 明确区分人工/AI 权限边界
3. **主动设计型** — 深度参与架构设计，而非被动接受
4. **结构化偏好** — 喜欢分层次、有明确格式的信息组织
